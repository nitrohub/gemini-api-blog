In this blog post, I'll walk you through building a Gemini API and integrating it with a CI/CD pipeline for testing and deployment. By the end, you'll understand how to use automation to streamline your development process.

Introduction

The Gemini API is a custom-built Flask API designed for generating text based on user prompts. The API leverages advanced language models and provides a user-friendly interface to customize instructions for the content it generates.

To ensure reliability and scalability, I implemented a CI/CD pipeline using GitHub Actions. This pipeline automates testing, building, and deployment, making the development process faster and more robust.

Setting Up the API

The Gemini API has several key functionalities:

  • Handling user prompts dynamically.
  • Providing customizable instructions to tailor generated content.
  • Integrating seamlessly with a Flask backend.

Here’s an example of the generate_content function that powers the API:

from flask import Flask, request, jsonify

app = Flask(__name__)

@app.route('/generate', methods=['POST'])
def generate_content():
    data = request.json
    prompt = data.get('prompt', '')
    instructions = data.get('instructions', 'Default instructions')
    # Generate content (mocked for simplicity)
    generated_content = f"Generated content based on: {prompt} with {instructions}"
    return jsonify({'content': generated_content})

Testing the API

Testing was critical to ensuring the Gemini API works as expected. Here’s how I approached it:

  • **Unit Tests**: Tested individual functions like generate_content.
  • **Integration Tests**: Verified API routes and responses.

Example test case:

import unittest
from app import app

class TestGeminiAPI(unittest.TestCase):
    def test_generate_content(self):
        with app.test_client() as client:
            response = client.post('/generate', json={
                'prompt': 'Hello, world!',
                'instructions': 'Make it friendly.'
            })
            self.assertEqual(response.status_code, 200)
            self.assertIn('content', response.json)

Setting Up CI/CD with GitHub Actions

The CI/CD pipeline automates the following tasks:

  1. Building Docker images for the API.
  2. Running all unit and integration tests.
  3. Deploying to a production server.

Here’s the GitHub Actions workflow file:

name: CI/CD Pipeline

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
    - name: Checkout Code
      uses: actions/checkout@v2

    - name: Set Up Python
      uses: actions/setup-python@v2
      with:
        python-version: '3.8'

    - name: Install Dependencies
      run: pip install -r requirements.txt

    - name: Run Tests
      run: pytest

  deploy:
    runs-on: ubuntu-latest
    needs: build
    steps:
    - name: Deploy to Production
      run: echo "Deploying to production..."
